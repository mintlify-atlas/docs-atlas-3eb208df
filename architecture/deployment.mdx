## Deployment Overview

CryptoTracker Web is deployed on Google Cloud Platform (GCP) using a multi-VM architecture. Each microservice runs on its own Compute Engine instance, with a shared MySQL database and NGINX-based frontend.

## Service Deployment

### Frontend Deployment (NGINX)

#### 1. Install NGINX
```bash
sudo apt-get update
sudo apt-get install -y nginx
```

#### 2. Configure NGINX

Create `/etc/nginx/sites-available/cryptotracker`:

```nginx
server {
    listen 80;
    server_name _;
    
    root /var/www/cryptotracker;
    index index.html;
    
    # Enable gzip compression
    gzip on;
    gzip_types text/css application/javascript text/html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    # Cache static assets
    location ~* \.(css|js|png|jpg|jpeg|gif|ico|svg)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
    
    # Proxy to backend services (if needed)
    location /api/precios {
        proxy_pass http://10.23.176.x:8080/precios;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /api/grafico {
        proxy_pass http://10.23.176.x:8081/grafico;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### 3. Deploy Frontend Files
```bash
# Create web root
sudo mkdir -p /var/www/cryptotracker

# Upload files using gsutil
gsutil cp gs://fontend-bucket/index.html /var/www/cryptotracker/
gsutil cp gs://fontend-bucket/style.css /var/www/cryptotracker/

# Or upload directly
sudo scp -i ~/.ssh/gcp-key index.html user@VM_IP:/var/www/cryptotracker/
sudo scp -i ~/.ssh/gcp-key style.css user@VM_IP:/var/www/cryptotracker/

# Set permissions
sudo chown -R www-data:www-data /var/www/cryptotracker
sudo chmod -R 755 /var/www/cryptotracker
```

#### 4. Enable Site
```bash
sudo ln -s /etc/nginx/sites-available/cryptotracker /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx
```

### MySQL Database Deployment

#### 1. Install MySQL Server
```bash
sudo apt-get update
sudo apt-get install -y mysql-server
```

#### 2. Secure Installation
```bash
sudo mysql_secure_installation
```

Configuration:
- Set root password: `root` (as per source code)
- Remove anonymous users: Yes
- Disallow root login remotely: No (needed for our setup)
- Remove test database: Yes
- Reload privilege tables: Yes

#### 3. Configure MySQL for Remote Access

Edit `/etc/mysql/mysql.conf.d/mysqld.cnf`:

```ini
[mysqld]
# Bind to all interfaces (for internal network access)
bind-address = 0.0.0.0

# Performance tuning
max_connections = 200
innodb_buffer_pool_size = 4G
innodb_log_file_size = 512M

# Enable SSL
require_secure_transport = ON
ssl-ca=/var/lib/mysql/ca.pem
ssl-cert=/var/lib/mysql/server-cert.pem
ssl-key=/var/lib/mysql/server-key.pem
```

#### 4. Create Database and Tables

```bash
sudo mysql -u root -p
```

```sql
-- Create database
CREATE DATABASE criptomonedas_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE criptomonedas_db;

-- Create tables for each cryptocurrency
CREATE TABLE bitcoin (
    id INT AUTO_INCREMENT PRIMARY KEY,
    precio DECIMAL(18,2) NOT NULL,
    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_fecha (fecha_registro)
) ENGINE=InnoDB;

CREATE TABLE ethereum LIKE bitcoin;
CREATE TABLE ripple LIKE bitcoin;
CREATE TABLE solana LIKE bitcoin;
CREATE TABLE tron LIKE bitcoin;
CREATE TABLE dogecoin LIKE bitcoin;
CREATE TABLE cardano LIKE bitcoin;
CREATE TABLE hyperliquid LIKE bitcoin;
CREATE TABLE bitcoin_cash LIKE bitcoin;
CREATE TABLE chainlink LIKE bitcoin;

-- Create user for remote access
CREATE USER 'root'@'10.23.176.%' IDENTIFIED BY 'root';
GRANT ALL PRIVILEGES ON criptomonedas_db.* TO 'root'@'10.23.176.%';
FLUSH PRIVILEGES;

-- Verify tables
SHOW TABLES;
```

#### 5. Restart MySQL
```bash
sudo systemctl restart mysql
sudo systemctl enable mysql
```

#### 6. Test Connection
```bash
# From another VM in the network
mysql -h 10.23.176.2 -u root -p -e "SHOW DATABASES;"
```

### Java Microservices Deployment

#### Prerequisites on All Java VMs

```bash
# Install Java 17
sudo apt-get update
sudo apt-get install -y openjdk-17-jdk

# Verify installation
java -version

# Create application directory
sudo mkdir -p /opt/cryptotracker
sudo chown $USER:$USER /opt/cryptotracker
```

#### WebServer Deployment

```bash
# Upload files
cd /opt/cryptotracker
mkdir -p webserver/lib

# Copy compiled class and libraries
scp WebServer.class user@VM_IP:/opt/cryptotracker/webserver/
scp lib/mysql-connector-java-*.jar user@VM_IP:/opt/cryptotracker/webserver/lib/

# Or compile on server
javac -cp ".:lib/*" WebServer.java
```

#### Create Systemd Service

Create `/etc/systemd/system/webserver.service`:

```ini
[Unit]
Description=CryptoTracker WebServer
After=network.target mysql.service

[Service]
Type=simple
User=cryptotracker
WorkingDirectory=/opt/cryptotracker/webserver
ExecStart=/usr/bin/java -cp ".:lib/*" WebServer 8080
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

# Resource limits
MemoryMax=512M
CPUQuota=50%

[Install]
WantedBy=multi-user.target
```

```bash
# Enable and start service
sudo systemctl daemon-reload
sudo systemctl enable webserver
sudo systemctl start webserver
sudo systemctl status webserver

# View logs
sudo journalctl -u webserver -f
```

## Security Hardening

### OS Security

```bash
# Update all packages
sudo apt-get update && sudo apt-get upgrade -y

# Enable automatic security updates
sudo apt-get install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades

# Configure firewall (ufw)
sudo ufw allow 22/tcp     # SSH
sudo ufw allow 80/tcp     # HTTP (frontend only)
sudo ufw allow 8080/tcp   # WebServer
sudo ufw allow 8081/tcp   # GraficoService

```

### MySQL Security

```bash
# Disable remote root login
mysql -u root -p
DELETE FROM mysql.user WHERE User='root' AND Host NOT IN ('localhost', '127.0.0.1', '::1');
FLUSH PRIVILEGES;

# Create limited user for applications
CREATE USER 'cryptoapp'@'10.23.176.%' IDENTIFIED BY 'secure_password';
GRANT SELECT, INSERT ON criptomonedas_db.* TO 'cryptoapp'@'10.23.176.%';
FLUSH PRIVILEGES;
```

### SSH Hardening

Edit `/etc/ssh/sshd_config`:

```bash
PermitRootLogin no

```

## Disaster Recovery

### Backup Strategy

#### Database Backups

```bash
# Create backup script
cat > /opt/backup-mysql.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/var/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

mysqldump -u root -proot criptomonedas_db | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Upload to Cloud Storage
gsutil cp $BACKUP_DIR/db_$DATE.sql.gz gs://cryptotracker-backups/mysql/

# Delete local backups older than 7 days
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete
EOF

chmod +x /opt/backup-mysql.sh

# Add to crontab (daily at 2 AM)
0 2 * * * /opt/backup-mysql.sh
```

#### Application Backups

```bash
# Snapshot all VM disks
gcloud compute disks snapshot DISK_NAME \
  --snapshot-names=DISK_NAME-$(date +%Y%m%d) \
  --zone=us-central1-a
```

### Recovery Procedures

#### Database Restore

```bash
# Download backup
gsutil cp gs://cryptotracker-backups/mysql/db_TIMESTAMP.sql.gz /tmp/

# Restore
gunzip /tmp/db_TIMESTAMP.sql.gz
mysql -u root -p criptomonedas_db < /tmp/db_TIMESTAMP.sql
```

#### Service Restore

```bash
# Restore from disk snapshot
gcloud compute disks create DISK_NAME-restored \
  --source-snapshot=SNAPSHOT_NAME \
  --zone=us-central1-a
```

| **Total** | **~$133-148/month** |

### Optimization Strategies

1. **Use Preemptible VMs** for non-critical services (70% cost reduction)
2. **Committed Use Discounts** (up to 57% off)
3. **Autoscaling** to scale down during low traffic
4. **Regional Storage** instead of multi-regional
5. **Compress Images** from GraficoService before transmission
